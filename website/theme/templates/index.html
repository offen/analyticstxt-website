A proposed standard
which allows websites and services to disclose information
about their usage of analytics software and user tracking.

Learn more~

Privacy has become an important feature for users of websites and services.

We aim at providing a well-defined way for websites and services to signal their usage of analytics and tracking software to users and tooling used by users.

analytics.txt will be designed to be an elaborate standard that describes the usage of analytics and tracking in a non-biased way that is understandable both for a non-technical audience as well as useful for consumption by tools and software.

First version draft in progress~

We are currently working on drafting a first version of the specification and plan to submit it as an IETF Internet Draft in the summer of 2021.

A first draft will be published here as soon as it's ready.

Why should I use analytics.txt on my website?

  1. Regulations around the handling of user data on websites are evolving quickly and mandate certain practices. An analytics.txt file can give auditors instant insights on what is in use and how this maps to the applicable regulations.

  2. Users value websites that are transparent about the privacy implications that using them brings. analytics.txt can disclose such information to your users without hiding behind the legalese walls of text that have become a practice.

  3. Tools like crawlers or Browser extensions can read and parse the data in analytics.txt and use it to signal privacy metrics to users and other third parties.

How a typical analytics.txt file looks like.

# analytics.txt file for https://www.analyticstxt.org

Author: Frederik Ring

Collects: url, referrer, device-type

Stores: first-party-cookies, local-storage

# Data is encrypted end-to-end

Uses: javascript

# Users can also delete their usage data only without opting out

Allows: opt-in, opt-out

# Data is retained for 6 months

Retains: P6M

# Optional fields

Session: user

Visibility: user

Compliance: gdpr

Vendors: offen

Where should I put an analytics.txt file?

The preferred default location for your analytics.txt file is the /.well-known/ path of your web server. If this is not ideal or possible for you, there are two alternatives:

  1. Use a <link rel="analytics.txt" href="/path/to/analytics.txt"> in the head of your document.

  2. Send an X-Analytics-Txt: https://www.example.com/path/to/analytics.txt header in your responses.

I would like to use analytics.txt but don't really know what to put in there?

We are working hard to make the specification as accessible as possible for anyone. For this purpose, we are currently working on an Implementation Guide and Vendor Presets that we plan to publish in the near future.

How do I know the information in an analytics.txt file is correct?

analytics.txt is designed in a way that there are no right or wrong answers, so we start by trusting others to populate it correctly. If you are unsure about the correctness of such a file, you can do two things:

  1. Contact the email address given in the mandatory Review field of the analytics.txt file, asking for clarification.

  2. Use the Audit Tool we will be providing to check for basic conformance.
